이번 편은 회고 같은 글이다. 며칠을 들여 만든 디자인 시스템을 *같은 주에* 은퇴시킨 이야기.

## 시작 — Apple 톤이 안 맞는다는 깨달음

처음의 `DESIGN.md`는 Apple.com 마케팅 페이지를 정밀 분석한 시스템이었다. SF Pro, 단일 액션 블루 `#0066cc`, 풀-블리드 라이트/다크 타일 교차, 극도로 비워진 UI 크롬. "박물관 갤러리" 톤.

이게 좋은 시스템이긴 하지만 `docs/adr/0004-pegman-brand-system.md` 에서 솔직히 적었다.

> Apple식 미니멀이 성립한 핵심 이유는 *제품 사진이 색·감정·내러티브를 모두 짊어졌기 때문*이다. 페그보드는 사진이 없는 ~100개 작은 유틸리티 페이지 — UI 자체가 브랜드를 짊어져야 한다.

페그보드는 "벽에 도구를 거는 작업대"라는 메타포를 갖고 출발한 사이트다 ([1편 참조](/blog/260609-pegboard-01-start)). 차가운 럭셔리 톤은 이 메타포와 충돌한다. 친근한 장인의 작업 공간이어야 한다.

그래서 결정했다 — **Apple식 시스템 전면 폐기, Pegman이라는 캐릭터 중심 시스템을 백지에서 재구축.**

## Pegman — 캐릭터 시스템

`docs/pegman_guide.png`에 캐릭터를 정의했다. 주황 안전모, 코발트 오버롤. 7개 포즈로 카테고리와 1:1 매핑.

| 카테고리 | Pegman 포즈   |
| -------- | ------------- |
| text     | 메모패드를 든 |
| calc     | 계산기를 든   |
| convert  | `</>` 표지판  |
| format   | 플로우차트    |
| time     | 시계          |
| audio    | (이후 추가)   |
| ai       | (이후 추가)   |

ADR-0004의 핵심 시스템 결정들:

- **이중 액센트, 역할 분리**: Safety Orange `#F88008` = 행동(CTA, 포커스, 활성). Cobalt `#1068C0` = 구조(네비, 링크). 같은 역할에 두 색을 함께 쓰지 않는다.
- **CTA 텍스트 = 네이비** (`#082048`, Pegman 외곽선 색). WCAG AAA 10.5:1. 건설 안전 표지판 idiom.
- **크림 캔버스 워크숍**: 페이지 바닥 `#F4ECD8` (따뜻한 페그보드 표면).
- **IBM Plex Sans + Plex Mono**: 7로케일 모두 공식 시블링 존재 (KR/JP/SC/TC).
- **페이지 해부 = 인사·비켜주기·돌아오기**: 히어로(Pegman 인사) → 도구 카드(Pegman 없음) → 빈상태/에러(Pegman 돌아옴). Clippy 함정 회피.
- **하이브리드 음성**: 히어로 부카피 + 상태 메시지만 Pegman 1인칭, 버튼·라벨은 중립.

이 시스템에는 야망이 있었다. **신규 유틸 추가 비용 일정** — 카테고리만 지정하면 마스터 포즈를 자동 상속해서 100페이지까지 확장해도 아트 작업이 0회.

## 5단계 점진 출하

빅뱅 마이그레이션은 너무 위험했다. 그래서 5단계로 갔다.

```
Phase 0: ADR-0004 + 새 DESIGN.md 작성
Phase 1: globals.css 토큰 시스템 (크림/오렌지/코발트)
Phase 2: 공유 프리미티브 + SVG 플레이스홀더
Phase 3: 페이지 통합 (페이지 1개당 1 PR, 사용자 지시 후)
   ├── 3-home, 3-base64, 3-json, 3-dummy-text, 3-character-count,
   ├── 3-timezone, 3-unix, 3-calc (3개 페이지 묶음)
Phase 4: 404 + 에러 상태 (PegmanState 카드) + 코발트 카테고리 strip + 7로케일 음성 카피
```

5월 25일 ~ 27일 사이에 진행됐다. 9개 페이지 × Pegman 통합 + 7로케일 음성 카피 + PWA 매니페스트 + 마스커블 아이콘.

이것만으로도 꽤 큰 작업이었다.

## 그리고 — ADR-0006 (shadcn 도입, 5월 30일)

페이지가 18개로 늘면서 "코발트 가로 strip" 기반 네비가 한계에 부딪쳤다. shadcn 문서처럼 *좌측 사이드바 카테고리 내비*로 옮기고 싶어졌다.

여기서 핵심 긴장이 생겼다 (`docs/adr/0006-shadcn-component-layer-sidebar-ia.md`).

> "shadcn 도입 = 디자인 언어 폐기"가 아니다. shadcn은 컴포넌트를 레포에 복사해 소유하는 방식이고, 테마는 CSS 변수 한 묶음이라 브랜드 색을 그 변수 슬롯에 부어넣으면 된다.

그래서 결정했다 — **브랜드(DESIGN.md/ADR-0004)는 SoT로 보존하고, shadcn은 컴포넌트·레이아웃 골격으로만 도입한다.**

크림/오렌지/코발트 + IBM Plex + Pegman은 유지. shadcn 변수(`--primary`, `--sidebar-*`)를 브랜드 토큰 위에 *alias*로 얹는다. 그리고 ADR-0006에 친절하게 **트랩 노트**까지 박았다.

> **주의(함정):** shadcn-create로 생성한 프리셋(`oklch` 블루 `--primary`, 순백 `--background`, 쿨 그레이 neutral)은 **스키마 템플릿으로만** 쓴다 — 변수 이름 목록과 라이트/다크 구조만 차용하고, **값은 채택하지 않는다.** 생성 프리셋의 블루 primary를 그대로 붙이면 CTA가 파랑이 되어 orange=액션 규칙이 깨진다 — 하지 말 것.

> "**하지 말 것.**"

이게 5월 30일이다.

## 5월 31일 — ADR-0007, 그 트랩을 *의도적으로* 넘는다

다음 날, 미래의 내가 그 트랩 노트를 넘었다 (`docs/adr/0007-pure-shadcn-preset-retire-pegman-brand.md`).

> 사용자가 그 거부했던 대안을 **의도적으로 채택**하기로 결정했다. 이유는 단순·일관성: brand 토큰 alias 층(방금 "레거시 Apple alias 제거" 커밋으로 3중 시스템을 막 해소한 직후)을 더 쌓지 않고, shadcn 표준 프리셋 한 벌만 남기는 단일 어휘를 원한다.

뒤집힌 결정들:

| ADR-0006 (5월 30일)                                  | ADR-0007 (5월 31일)                                   |
| ---------------------------------------------------- | ----------------------------------------------------- |
| 브랜드 토큰 SoT 보존, shadcn은 alias로만             | shadcn 프리셋이 유일한 SoT                            |
| Orange `--primary` (액션) / Cobalt (구조) dual       | Blue `--primary` 단일                                 |
| IBM Plex Sans + Plex Mono + Bricolage 디스플레이     | system-ui / shadcn 기본 스택                          |
| Pegman 캐릭터 노출                                   | **Pegman 미노출** (삭제는 아님)                       |
| 크림 캔버스 `#F4ECD8` + 코발트 strip                 | shadcn stone 뉴트럴 + 블루 primary                    |

## "삭제"가 아니라 "미노출"

이 결정의 가장 흥미로운 디테일이다. ADR-0007은 Pegman을 *삭제하지 않는다.*

> **Pegman 미노출(삭제 아님).** `PegmanHero`/`PegmanState`/`PegmanLogo`/카테고리 마크 *렌더 호출만 제거*해 화면에서 안 보이게 한다. 컴포넌트 파일·`/public/pegman` 에셋·7로케일 음성 카피 i18n 키는 **보존**(되돌리기 쉽게, messages 광범위 수정 회피).

이게 영리한 부분이다. 마음이 또 바뀌어서 Pegman을 부활시키고 싶어지면, 렌더 호출만 다시 넣으면 된다. 컴포넌트 파일, 에셋, 7로케일 음성 카피는 그대로 자고 있다.

## 토큰 마이그레이션 — 치환 안 함

기술적으로도 비슷한 선택을 했다. 브랜드 클래스 (`bg-canvas`, `text-ink`, `text-cobalt` 등)가 코드에 ~400회 / 48파일에 박혀 있었다. 이걸 다 `bg-background` 같은 shadcn 클래스로 치환하는 게 정공법이지만, 치환 리스크와 검증 부담이 컸다.

대신 **alias 층**을 깔았다.

```css
:root {
  /* shadcn 프리셋 토큰 */
  --background: oklch(...);
  --primary: oklch(...);

  /* 브랜드 토큰은 shadcn 변수를 가리키는 alias로 보존 */
  --canvas: var(--background);
  --cobalt: var(--primary);
  --ink: var(--foreground);
  /* ... */
}
```

`@theme inline`도 shadcn 슬롯 + 기존 브랜드 `--color-*` 매핑을 *둘 다* 유지. 그래서 기존 유틸 클래스 `bg-canvas`/`text-ink`/`text-cobalt`가 그대로 컴파일되며 *렌더만 shadcn 값으로 바뀐다.* alias는 `var()` 런타임 해석이라 `.dark`는 shadcn 변수만 덮으면 자동으로 따라온다.

대가는 분명하다.

> 브랜드 클래스 이름이 소스에 잔존(어휘 2개 공존, alias 층). 화면은 100% shadcn.

이 트레이드오프가 옳은지는 시간이 알려줄 거다. 지금 시점에선 *되돌리기 비용을 낮게* 유지하는 게 더 중요하다고 판단했다.

## 잃어버린 것 — 솔직한 회고

ADR-0007의 Consequences에 솔직하게 적어둔 한 줄이다.

> **dual-accent·WCAG 문서·캐릭터 모트 상실.** ADR-0004가 차별화 근거로 든 "따뜻한 craftsman 톤 = 카테고리 모트"가 사라지고, dev-tool 표준 블루로 회귀. 사용자가 단순·일관성을 우선해 수용한 트레이드오프.

이게 진짜 잃은 것이다. dev-tool 카테고리(JSON formatter, base64, ...)는 거의 다 파랑 액센트의 무미건조 사이트다. Pegman + 따뜻한 팔레트는 즉각적인 카테고리 *모트(moat)*였다. 그걸 포기했다.

대신 얻은 것 — **단일 어휘**. 토큰이 한 벌, 폰트가 한 벌, 컴포넌트가 한 벌. 새 페이지를 만들 때 "이 색은 어디서 가져오지" 가 사라진다.

## 6일짜리 브랜드의 교훈

5월 25일 ADR-0004 → 5월 31일 ADR-0007. 정확히 일주일이다.

만든 것을 같은 주에 버리는 결정은 보통 *낭비*로 느껴진다. 그런데 다르게 보면 — **트레이드오프의 비용을 직접 만져보고 결정했다.** Pegman을 만들기 전에는 "캐릭터 시스템 vs 단일 프리셋"의 비용 비교가 가설이었다. 만들고 나니 그게 실제 코드와 PR 카운트로 보였다. 그리고 *이건 내가 굴리고 싶은 시스템이 아니다*가 명확해졌다.

ADR-0007의 사인오프에 한 줄로 적혀 있다.

> Status: Accepted (사용자 사인오프 — "아주 파괴적인 작업" 명시)

"아주 파괴적인 작업"을 인지하고 채택한다는 명시. 이게 ADR이 미래에 남기는 가장 중요한 신호다 — *내가 이 결정의 비용을 알고 있었다.*

## 정리

- 처음에는 Apple 톤이었고, 그게 안 맞는다는 깨달음으로 Pegman 캐릭터 중심 브랜드 시스템(ADR-0004)을 5단계로 출하했다.
- 5월 25~27일에 9개 페이지 + 7로케일 음성 카피 + PWA까지 완료.
- 5월 30일 ADR-0006으로 shadcn을 컴포넌트 레이어로만 도입 (브랜드 토큰은 보존). 트랩 노트로 "shadcn 블루 primary 그대로 쓰지 말 것" 박아둠.
- 5월 31일 ADR-0007로 그 트랩을 의도적으로 넘음. Pegman 미노출(삭제 X) + alias 층으로 브랜드 클래스 잔존 허용 + 순수 shadcn 프리셋이 단일 SoT.
- **삭제가 아니라 미노출** — 마음이 바뀌면 렌더 호출만 다시 넣으면 부활 가능. 되돌리기 비용을 낮게.
- 잃은 것: 카테고리 모트, 따뜻한 craftsman 톤. 얻은 것: 단일 어휘 — 토큰·폰트·컴포넌트가 한 벌.
- 6일짜리 브랜드의 교훈: **만들기 전에는 가설, 만들고 나면 비용이 보인다.** 만든 것을 버리는 결정은 낭비가 아니다.

다음 편은 무거운 유틸의 첫 사례 — **브라우저에서 ffmpeg 를 돌린 오디오 컨버터** 얘기다.
